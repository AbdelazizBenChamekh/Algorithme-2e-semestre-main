В этой лабораторной работе я реализовывал алгоритмы поиска и анализа подстрок.

**Задание 1 — Наивный поиск подстроки** *(дополнительное)*  
Нужно найти все вхождения шаблона в тексте. Я сдвигал окно шириной `|p|` по тексту и на каждой позиции сравнивал срез с шаблоном. Простой подход без предобработки.

**Задание 2 — Наибольшая палиндромная подпоследовательность**  
Нужно найти длину наибольшей подпоследовательности, которая читается одинаково в обе стороны. Я строил 2D ДП-таблицу по длинам подстрок: если крайние символы совпадают — расширяем, иначе берём лучший из двух вариантов без одного из краёв.

**Задание 6 — Z-функция**  
Нужно вычислить Z-массив строки, где `z[i]` — длина наибольшего префикса строки, совпадающего с подстрокой начиная с позиции `i`. Я использовал Z-алгоритм с поддержкой Z-окна `[l, r]`: для позиций внутри окна переиспользую уже вычисленные значения, для позиций вне — расширяю наивно.

**Задание 7 — Наибольшая общая подстрока**  
Нужно найти наибольшую непрерывную подстроку, входящую в обе строки, и вернуть её позиции. Я строил 2D ДП-таблицу: `dp[i][j]` — длина общего суффикса `p[0..i]` и `t[0..j]`. При совпадении символов продолжал цепочку, при несовпадении — сбрасывал в 0. Отслеживал максимум и позицию его окончания.
